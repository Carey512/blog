DigitalOcean 购买海外服务器流程

从准备 SSH 公钥、选择 Droplet 配置、绑定支付方式到拿到公网 IP 的完整购买步骤，适合第一次部署博客项目时照着执行。

这篇文档只记录购买服务器阶段，不包含后续 Docker 安装和项目部署。当前项目建议先买一台海外 Ubuntu 服务器，用公网 IP 访问，等项目稳定后再绑定域名和 HTTPS。

推荐先用最低配或入门配置完成第一版部署验证。博客项目成熟后，再按访问量、图片/音乐资源体积、后台上传频率逐步升级服务器或拆分对象存储。

DigitalOcean 的界面和支付能力可能随账号地区变化，付款方式以结账页面实际显示为准；如果看不到本地支付方式，优先准备国际信用卡、借记卡、PayPal、Google Pay 或 Apple Pay 等可用方式。

01

准备 SSH 公钥

  • 在 PowerShell 新窗口执行 ssh -V，确认 OpenSSH 已经加入 Path。
  • 如果还没有密钥，执行 ssh-keygen -t ed25519 -C "blog-server" 生成密钥。
  • 查看公钥命令：Get-Content $env:USERPROFILE\.ssh\id_ed25519.pub。
  • 只复制 .pub 公钥内容，不要复制没有 .pub 后缀的私钥文件。
02

进入 DigitalOcean 控制台

  • 打开 https://cloud.digitalocean.com/ 并注册或登录账号。
  • 如果是新账号，先完成邮箱验证和基础账号信息。
  • 进入 Create 菜单，选择 Droplets；也可以直接打开 https://cloud.digitalocean.com/droplets/new。
  • 项目可以先放在默认 first-project，后续再按环境拆分 production、staging。
03

选择机房区域

  • 优先选择离目标用户更近的海外节点，例如 Singapore、San Francisco、New York、Frankfurt。
  • 如果你的主要访问者在国内，Singapore 通常延迟更友好。
  • 先不要为延迟纠结太久，第一台机器重点是跑通部署闭环。
  • 后续有真实访问量后，再用监控和测速决定是否迁移区域。
04

选择系统镜像

  • Image 选择 Ubuntu LTS 版本，优先选择页面推荐的长期支持版本。
  • 博客项目后续会用 Docker 部署，所以不需要选择 Node.js、Nginx 等 Marketplace 镜像。
  • 保持系统尽量干净，部署依赖统一交给 Docker 和 docker compose。
  • 创建后记录系统版本，方便后续写部署文档和排错。
05

选择服务器规格

  • 第一版可以选择 Basic / Regular / 1 vCPU / 1GB RAM / 25GB Disk 这类入门配置。
  • 如果后台上传图片、音乐文件较多，磁盘和对象存储会比 CPU 更早成为关注点。
  • 生产成熟后建议升级到至少 2GB RAM，构建和运行会更稳。
  • 购买时确认右侧费用预估，避免误选高配或附加服务。
06

添加 SSH Key

  • Authentication Method 选择 SSH Key，不建议第一次就只用密码登录。
  • 点击 Add SSH Key，把本地 .pub 公钥完整粘贴进去。
  • Key Name 可以填写 blog-server、home-pc 或当前电脑名称，方便以后识别。
  • 保存后勾选这把 key，确保新 Droplet 创建时自动写入服务器。
07

设置名称、标签和项目

  • Droplet name 建议使用 blog-prod-01，名称要能看出项目、环境和编号。
  • Tags 可以添加 blog-prod-01、blog、production，方便后续筛选资源。
  • Quantity 保持 1，第一阶段不需要多台服务器。
  • 确认 Project 选择正确，避免资源散落在不同项目里。
08

绑定支付方式并创建

  • 右侧如果提示 Add Payment Method and Create Droplet，需要先添加支付方式。
  • DigitalOcean 官方列出信用卡、借记卡、PayPal、Google Pay、Apple Pay、Alipay、Link 等方式，实际可用项以当前账号结账页为准。
  • 支付方式添加成功后，再点击 Create Droplet 开始创建服务器。
  • 创建过程中不要重复点击，等待控制台显示 Droplet 状态和公网 IP。
09

保存服务器关键信息

  • 复制服务器公网 IPv4，后续访问和 SSH 都会用到。
  • 记录服务器名称、区域、系统、规格、创建时间和月费用。
  • 建议把这些信息写回本项目文档，形成自己的部署资产清单。
  • 如果后续买域名，公网 IP 会用于配置 DNS A 记录。
10

首次 SSH 连接验证

  • 在 PowerShell 执行 ssh root@服务器IP。
  • 第一次连接出现 yes/no 提示时，输入 yes 并回车。
  • 如果提示权限或密钥问题，检查 DigitalOcean 是否勾选了正确 SSH Key。
  • 能进入 root@ubuntu 提示符，就说明购买和登录阶段完成。
11

购买完成后的下一步

  • 更新系统包，安装 Docker Engine 和 Docker Compose 插件。
  • 从 GitHub 拉取 E:\blog 对应仓库代码。
  • 配置生产环境变量，再执行 docker compose up -d --build。
  • 先用 http://服务器IP/、/admin/、/health 验证，再考虑域名、HTTPS、备份和监控。
