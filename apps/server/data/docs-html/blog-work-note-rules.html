工作笔记沉淀规则

把临时经验整理成可复用文档，而不是散落在聊天记录里。

每篇文档只解决一个主题，例如部署、快捷操作、账号权限、接口说明，不要把多个问题混在一起。

文档标题尽量使用动作化表达，例如“如何部署项目”比“部署”更容易搜索。

每次完成一次真实操作后，把命令、报错、解决方式和下一步优化写回文档。

如果一篇文档开始变长，就拆成主文档和子文档，主文档保留流程，子文档记录细节。

后续 admin 端可以增加编辑、标签管理和版本记录，让这个模块变成个人知识库。
